Multi-Modal Robotic Gantry

A Real-Time Vision-Based Target Practice System

Project Overview

This project is a 2-axis robotic gantry system designed for ping-pong target practice. It integrates a real-time vision system for automated ball tracking and interception, alongside multiple manual control options.

The core objective was to create a versatile and responsive robotic system by interfacing various sensors, controllers, and microcontrollers, bridging the gap between high-level computer vision and low-level hardware control. The system is built on an 80/20 aluminum frame, driven by stepper motors, and controlled by an STM32 microcontroller.

Key Features

Automated Control

Real-time ball tracking using a Raspberry Pi with OpenCV.

Manual Control

User-operated control via a PS2 joystick or a custom wireless Wii remote.

Embedded System Core

STM32 microcontroller manages motor control, sensor data, and communication protocols.

Technical Implementation

Vision System (Raspberry Pi)

A Raspberry Pi 4 is dedicated to the computer vision task. A Python script leverages the OpenCV library to continuously capture video frames, convert them to the HSV color space for robust detection, apply a color mask to isolate the ping-pong ball, and calculate its centroid coordinates (X, Y). These coordinates are then transmitted to the STM32 controller over SPI.

Firmware & Control (STM32)

The STM32 runs C-based firmware and acts as the central brain. It features a state machine to switch between control modes (Automated, Wii, PS2). It receives data from the Raspberry Pi (SPI) and the custom Wii remote (I2C/UART) and translates it into precise PWM signals for the stepper motor drivers. Interrupt-driven routines for limit switches ensure the gantry operates within safe physical boundaries.

Below is a snippet from mainstepper.c, which is the core firmware running on the STM32. This section shows the main loop, which handles the state machine for switching between control modes and processing the corresponding inputs.

int main(void)
{
  // ... (HAL Init, Clock Config, Peripheral Init) ...

  printf("System Initialized. Entering main loop.\\n\\r");

  while (1)
  {
    // --- Check for mode switch button press ---
    GPIO_PinState current_button_state = HAL_GPIO_ReadPin(MODE_BUTTON_PORT, MODE_BUTTON_PIN);
    if (current_button_state == GPIO_PIN_SET && last_button_state == GPIO_PIN_RESET) {
        if (!button_pressed_flag) {
            button_pressed_flag = 1;
            
            // Cycle through control modes
            current_control_mode = (ControlMode_t)(((int)current_control_mode + 1) % (int)NUM_CONTROL_MODES);
            display_show_mode(current_control_mode);
            
            // Stop motors and handle de-initialization/re-initialization of peripherals
            // ...
        }
    }
    last_button_state = current_button_state;


    // --- Process Input Based on Current Mode ---
    switch(current_control_mode) {
        case CONTROL_MODE_OPENCV:
            if (new_spi_data_ready) {
                process_spi3_data_direct_control();
                new_spi_data_ready = 0;
                // Restart SPI DMA for next transaction
            }
            break;

        case CONTROL_MODE_PS2:
            handle_ps2_control();
            break;
            
        case CONTROL_MODE_USART2: // Wii Remote
            handle_usart2_control();
            break;
    }

    HAL_Delay(1);
  }
}
